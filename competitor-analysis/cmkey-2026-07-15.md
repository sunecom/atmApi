## 一、核心发现：套皮 Claude

**结论**：cmkey.cn 的"GLM-5.2"实际上是 **Claude 模型套皮**，不是真正的智谱 GLM。

### 证据链

1. **usage 字段暴露真相**：
   ```json
   {
     "usage_source": "anthropic",
     "claude_cache_creation_5_m_tokens": 0,
     "claude_cache_creation_1_h_tokens": 0
   }
   ```
   - `usage_source: "anthropic"` — 明确标识上游是 Anthropic
   - `claude_cache_creation_*` — Claude 特有的缓存字段

2. **模型自相矛盾**：
   - 模型自称"深度求索的 glm-5.2"
   - 但 usage 显示上游是 Anthropic Claude
   - 说明 system prompt 被注入了身份伪装

3. **响应 ID 格式异常**：
   - 正常智谱：`chatcmpl-xxxx` 或类似格式
   - cmkey：`cht000ddf34@dx19f639a10442871503` — 自定义格式

## 三、上游推测

**真实上游**：Anthropic Claude（大概率 claude-3.5-sonnet 或 claude-3-haiku）

**推测依据**：
1. usage 字段明确标注 `anthropic`
2. Claude 缓存字段完整暴露
3. 响应速度 2.2s 符合 Claude 海外 API 延迟
4. 不支持多模态 — 可能是故意阉割，也可能是 Claude 版本限制

**商业模式**：
- 用 Claude API 成本（约 $3/M input, $15/M output）
- 包装成"GLM-5.2"卖给国内用户
- 99元/月 = 约 $14，如果用户用量不大，有利润空间
- 1000次/5小时限流控制成本

## 四、目标客户

### 第一梯队：OpenClaw 用户（最快成交）

**痛点**：官方套餐抢不到、限速、按 Token 失控、GLM 空响应、配置失效

**适合**：deepseek-a4 基础/专业版、GLM-5.2 体验版

**获客**：故障诊断 + 配置支持

### 第二梯队：AI 编程重度用户（高价值）

**包括**：Cursor/Claude Code/Codex 用户、独立开发者、软件外包

**痛点**：长上下文、工具调用、月度成本可控、官方限速备选

**适合**：deepseek-a4 专业/创业版、GLM-5.2 标准/专业版

**获客**：技术文章 + B 站视频

### 第三梯队：渠道合作伙伴（杠杆最大）

**包括**：OpenClaw 部署商、教程作者、AI 工具社群、VPS 代理

**合作**：邀请码 + 续费佣金（首单 20%、续费 10%、30 天后结算）

### 暂不重点拓展

- 完全不懂 API 的泛消费者
- 需要完整 SSO/发票/SLA 的大企业
- 只找无限量/共享 Key 的套利用户

## 六、成交漏斗

```
流量 → 免费配置诊断 → 20 次试用 → 首次接入成功 → 付费 → 续费
```

### 6.1 免费配置诊断（引流钩子）

用户填写：使用的客户端、每天任务量、是否需要图片/长上下文、当前费用、常见错误

输出：推荐套餐 + 档位 + 预计月成本 + 可复制配置

### 6.2 小额试用

- 20 次基础调用，一个账号只领一次
- 高输出/超长上下文/最高 reasoning 不进入试用
- 试用用于验证接入，不是免费完成生产任务

### 6.3 首次接入成功保障

- 购买后提供一次 OpenClaw 配置协助
- 按文档和客服指导仍无法完成首次调用，可申请退款

### 6.4 公开信任页

官网必须公开：服务状态、30 天成功率、P50/P95 响应时间、维护公告、计费公式、退款规则

## 九、避免营销亏损

**核心原则**：利润来自未用满、路由、缓存和优化，不是 Token 加价。

**必须执行**：
- 不宣传无限量
- 清楚展示 5h/日/月限额
- 使用达到 70%/85%/95% 时提醒
- 重度用户主动推荐升级
- 对共享/转售/批量注册/自动刷量进行风控
- 按获客批次计算真实毛利
- 渠道佣金与 30 日留存挂钩
- 每周核对高消耗用户成本

**可承受 CAC**：
```
可承受 CAC = (90 日续费收入 - 90 日上游成本 - 支付退款 - 客服佣金 - 目标利润) / 用户数
```

没有足够队列数据前，不进行大规模付费投放。

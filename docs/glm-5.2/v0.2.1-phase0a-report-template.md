## 3. 口径说明

- high≈80%、xhigh≈95% 只是事前规划估算，不是 GLM-5.2 服务承诺。
- `ttft_ms` 只有流式用例表示真实首个 SSE data event；非流式标记为 `full_response`。
- `first_reasoning_ms` 与 `first_visible_ms` 分别记录首个推理事件和首个用户可消费结果，二者不能混为同一个 TTFT。
- `visible_tokens = completion_tokens - reasoning_tokens` 是 usage 口径推导值。
- `observed_terminal_kind` 根据 `content/tool_calls/refusal/reasoning_only/empty` 汇总黑盒证据，但 Task 0A 不替代 Task 4 的正式终态分类器。
- `estimated_worst_cost_usd` 只用于运行前费用护栏，最终成本以 `usage.cost` 或 OpenRouter 账单为准。
- 同一 history/scenario 使用稳定的合成 `session_id`，便于观察 cached token；它不包含真实用户身份。
